# 2. Начало работы

Это руководство поможет вам настроить и запустить проект **AIRBRO Business** локально для разработки.

## Шаг 1: Предварительные требования

Убедитесь, что на вашем компьютере установлены:

- **Node.js:** версия `20.x` или выше.
- **npm:** версия `10.x` или выше (обычно поставляется с Node.js).
- **Git:** система контроля версий.

## Шаг 2: Клонирование репозитория

Откройте терминал и выполните следующую команду:

```bash
git clone https://github.com/Sskutushev/AIRBRO-Business.git
cd AIRBRO-Business
```

## Шаг 3: Настройка бэкенда

Бэкенд требует отдельной настройки.

1.  **Перейдите в директорию бэкенда:**

    ```bash
    cd backend
    ```

2.  **Установите зависимости:**

    ```bash
    npm install
    ```

3.  **Настройте переменные окружения:**
    - Создайте файл `.env` в папке `backend`.
    - Скопируйте в него базовую конфигурацию:
      ```env # URL для подключения к базе данных. Для локальной разработки используется SQLite.
      DATABASE_URL="file:./dev.db"

          # Секретный ключ для подписи JWT-токенов. Придумайте любую сложную строку.
          JWT_SECRET="your-super-secret-jwt-key"

          # URL фронтенда для настройки CORS
          FRONTEND_URL="http://localhost:5173"

          # Настройки логирования (опционально)
          LOG_LEVEL="info"
          ```

      > **Важно:** `JWT_SECRET` должен быть надежным и уникальным.

4.  **Создайте и примените миграции базы данных:**
    Эта команда создаст файл базы данных `dev.db` и настроит все таблицы согласно схеме в `prisma/schema.prisma`.
    ```bash
    npx prisma migrate dev
    ```

## Шаг 4: Настройка фронтенда

Теперь настроим фронтенд.

1.  **Вернитесь в корневую директорию проекта:**

    ```bash
    cd ..
    ```

2.  **Установите зависимости:**

    ```bash
    npm install
    ```

3.  **Настройте переменные окружения (опционально):**
    - Фронтенд может работать без `.env` файла, используя моковые данные или подключаясь к локальному бэкенду по умолчанию.
    - Для подключения к продакшн-API или другим сервисам (Sentry, Google Analytics) создайте файл `.env` в корне проекта и добавьте соответствующие переменные:
      ```env
      VITE_API_URL="http://localhost:3000"
      VITE_SENTRY_DSN="..."
      VITE_GA_MEASUREMENT_ID="..."
      ```

## Шаг 5: Запуск проекта

Теперь, когда все настроено, можно запустить оба сервера.

1.  **Запустите бэкенд:**
    - Откройте **первый терминал**.
    - Перейдите в папку `backend` и запустите dev-сервер:
      ```bash
      cd backend
      npm run dev
      ```
    - Сервер запустится на `http://localhost:3000`.

2.  **Запустите фронтенд:**
    - Откройте **второй терминал**.
    - Находясь в корневой папке проекта, запустите dev-сервер:
      ```bash
      npm run dev
      ```
    - Веб-приложение откроется в браузере по адресу `http://localhost:5173`.

**Поздравляем!** Теперь у вас есть полностью рабочая локальная копия проекта AIRBRO Business.
