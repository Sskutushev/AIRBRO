# План доработки проекта AIRBRO Business

## Текущий статус:
- **Frontend (Лендинг):** Полностью реализован. Все 12 секций, дизайн-система, анимации, адаптивность и локализация (RU/EN) готовы.
- **Backend и API:** Полностью отсутствует. Функции аутентификации, платежей, отправки форм и основная бизнес-логика являются заглушками.

## План дальнейшей реализации:

### Этап 1: Реализация отправки формы обратной связи в Telegram

**Цель:** Сделать так, чтобы заявки с формы "CTA Section" на лендинге приходили в виде уведомлений в Telegram-чат.

1.  **Настройка окружения:**
    *   Создать файл `.env` в корне проекта для безопасного хранения секретных ключей (токен бота, ID чата).
    *   Получить от заказчика токен Telegram бота и ID чата для уведомлений.

2.  **Реализация Backend (`api/telegram.ts`):**
    *   Переписать существующую Vercel-функцию.
    *   Добавить логику для чтения переменных окружения (`process.env`).
    *   Реализовать функцию, которая принимает POST-запрос с данными формы.
    *   Форматировать полученные данные в читабельное сообщение.
    *   Использовать `fetch` для отправки этого сообщения в указанный Telegram-чат через `api.telegram.org`.
    *   Обеспечить обработку ошибок и возвращать корректные статусы (200 для успеха, 500 для ошибок).

3.  **Доработка Frontend (`src/components/sections/CTASection.tsx`):**
    *   Модифицировать функцию `handleSubmit`.
    *   Заменить текущую логику на `fetch`-запрос к эндпоинту `/api/telegram`.
    *   Реализовать управление состоянием отправки (`isSubmitting`, `isSuccess`, `isError`).
    *   Показывать пользователю индикатор загрузки на кнопке во время отправки.
    *   В случае успеха, отображать модальное окно с сообщением об успешной отправке.
    *   В случае ошибки, выводить сообщение об ошибке.

4.  **Тестирование:**
    *   Провести сквозное тестирование: заполнить форму на сайте и убедиться, что уведомление с корректными данными приходит в Telegram-чат.
    *   Проверить обработку ошибок (например, при неверном токене).
